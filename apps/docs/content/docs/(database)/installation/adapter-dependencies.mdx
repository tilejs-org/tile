# Dependências por adapter

O pacote principal resolve apenas o núcleo do Tile Database e o adapter local. Para adapters remotos, o projeto consumidor precisa fornecer as dependências e a conexão correspondente.

## Visão rápida

| Cenário | Pacotes extras | Import do adapter |
| --- | --- | --- |
| BSON local | nenhum | opcional, pois o local já é o padrão |
| MongoDB | `mongodb` | `@tile.js/database/adapters/mongodb` |
| Mongoose | `mongoose` | `@tile.js/database/adapters/mongoose` |

## Adapter padrão local

O adapter local pode ser usado sem nenhum import extra:

```ts
import { Database } from "@tile.js/database";

const database = new Database({ dbName: "app" });
```

Se você quiser instanciá-lo explicitamente, o `defaultAdapter` sai do entrypoint principal:

```ts
import { Database, defaultAdapter } from "@tile.js/database";

const database = new Database({
  storage: defaultAdapter({
    dbName: "app",
    storage: "workspace",
  }),
});
```

<Callout type="warning" title="Regra importante">
Quando `storage` recebe um adapter customizado, não passe `dbName` nem `compression` no `new Database(...)`. Essas opções pertencem ao próprio adapter.
</Callout>
