# Instalação

O `@tile.js/database` já inclui o núcleo da biblioteca e o adapter local em BSON.

Você só precisa instalar dependências extras se quiser trabalhar com **MongoDB** ou **Mongoose**.

## Pacote principal

<Tabs items={["bun", "npm", "pnpm", "yarn"]}>
  <Tab value="bun">

```bash
bun add @tile.js/database
```

  </Tab>

  <Tab value="npm">

```bash
npm install @tile.js/database
```

  </Tab>

  <Tab value="pnpm">

```bash
pnpm add @tile.js/database
```

  </Tab>

  <Tab value="yarn">

```bash
yarn add @tile.js/database
```

  </Tab>
</Tabs>

## Perfis de instalação

<Cards cols={3}>
  <Card title="BSON local" href="/docs/adapters/default">
    Nenhuma dependência extra. Ideal para começar rápido, CLIs e armazenamento local.
  </Card>

  <Card icon={<DatabaseIcon />} title="MongoDB" href="/docs/adapters/mongodb">

  </Card>

  <Card icon={<BoxesIcon />} title="Mongoose" href="/docs/adapters/mongoose">

  </Card>
</Cards>

## Requisitos

| Requisito | Versão recomendada |
| --- | --- |
| Node.js | `>= 24.x` |
| Bun | `>= 1.x` |
| TypeScript | `>= 5.x` |

## Verificando a instalação

```ts title="index.ts"
import { Database, Schema } from "@tile.js/database";

console.log("Tile Database pronto para uso");
```

<Tabs items={["bun", "tsx"]}>
  <Tab value="bun">

```bash
bun index.ts
```

  </Tab>

  <Tab value="tsx">

```bash
npx tsx index.ts
```

  </Tab>
</Tabs>

Se o arquivo executar sem erro de importação, seu ambiente já está pronto para seguir.

<Callout type="tip">
No adapter padrão, a pasta `.tile` só é criada quando a aplicação realmente precisa persistir dados ou metadados de collection.
</Callout>
