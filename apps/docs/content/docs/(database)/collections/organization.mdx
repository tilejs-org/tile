# Organização da camada de dados

A melhor estrutura é a menor estrutura que ainda deixa seu projeto claro.

## Padrões recomendados

<Tabs items={["simples", "modular", "monorepo"]}>
  <Tab value="simples">

<Files>
  <Folder name="src" defaultOpen>
    <Folder name="database" defaultOpen>
      <File name="index.ts" />
      <Folder name="schema" defaultOpen>
        <File name="member.ts" />
      </Folder>

    <File name="index.ts" />

É o padrão sugerido pela documentação: `src/index.ts` usa a database, `src/database/index.ts` centraliza a instância e `src/database/schema/member.ts` guarda a interface com o schema.

## Regras práticas

- mantenha `Schema` perto do domínio que ele representa
- exponha collections prontas em um ponto central (`src/database/index.ts` ou módulo equivalente)
- evite espalhar `new Database(...)` por vários arquivos
- concentre regras de negócio em serviços, não dentro dos schemas

<Callout type="tip">
Se você ainda está começando, prefira o padrão **simples**. Ele já cobre muito bem a maioria dos projetos e combina com o fluxo sugerido nesta documentação.
</Callout>
