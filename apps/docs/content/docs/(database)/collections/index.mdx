# Collections

Uma collection é a interface usada pela aplicação para trabalhar com documentos.

Toda collection pertence a uma instância de `Database` e utiliza um `Schema` para validar e normalizar os dados.

## Registrando uma collection

Normalmente, ela fica centralizada em `src/database/index.ts`:

```ts title="src/database/index.ts"
import { Database } from "@tile.js/database";
import { memberSchema, type Member } from "./schema/member.ts";

const connection = new Database({ dbName: "app" });

export const database = Object.assign(connection, {
  member: connection.collection<Member>("member", memberSchema),
});
```

`model()` é apenas um alias de `collection()`:

```ts
const member = connection.model<Member>("member", memberSchema);
```

## Cache por nome

```ts
const memberA = connection.collection("member", memberSchema);
const memberB = connection.collection("member", memberSchema);

console.log(memberA === memberB); // true
```

<Callout type="tip">
Cada instância de `Database` mantém uma única collection por nome. Isso evita recriação desnecessária e mantém a aplicação trabalhando sempre com o mesmo objeto.
</Callout>
