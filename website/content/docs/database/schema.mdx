# Schema

O `Schema` define a estrutura dos documentos de uma collection.

Além dos tipos dos campos, ele controla:

- validações
- defaults
- índices únicos
- `timestamps`
- `versionKey`
- metadados expostos via `describe()`

---

## Criando um Schema

```ts title="src/schema.ts"
import { Schema } from "@tile.js/database";

interface Member {
  _id?: string;
  name: string;
  email: string;
  age?: number;
  verified?: boolean;
}

export const memberSchema = new Schema<Member>({
  name: {
    type: String,
    required: true,
    trim: true,
  },
  email: {
    type: String,
    required: true,
    unique: true,
    lowercase: true,
  },
  age: {
    type: Number,
    min: 0,
  },
  verified: {
    type: Boolean,
    default: false,
  },
});
```

## Opções do Schema

```ts
const schema = new Schema(
  {
    name: String,
  },
  {
    timestamps: true,
    versionKey: true,
    _id: true,
    collection: "members",
  },
);
```

## Campos internos

Por padrão, o Tile.JS adiciona automaticamente:

| Campo | Descrição |
| ------ | --------- |
| `_id` | Identificador único do documento. |
| `__v` | Controle interno de versão. |
| `createdAt` | Data de criação (quando `timestamps` está ativo). |
| `updatedAt` | Data da última atualização (quando `timestamps` está ativo). |

## Índices únicos

Todo campo definido com `unique: true` recebe um índice próprio no adapter padrão local.

```ts
email: {
  type: String,
  unique: true,
}
```

Isso gera:

<Files>
  <Folder name="members" defaultOpen>
    <File name="_collection_index_email.json" />
  </Folder>
</Files>

Conteúdo:

```json title="_collection_index_email.json"
{
  "john@example.com": "68526dba05db8ec36dcb33d8"
}
```

<Callout type="tip">
Nos adapters de MongoDB e Mongoose, o índice é criado no backend remoto em vez de um arquivo local.
</Callout>

## Sobre validação

A validação é simples e previsível:

- filtros e tipos são rasos
- `Object` e `Array` não possuem schema aninhado
- `Date` aceita `Date`, `string` ou `number`, desde que o valor seja convertível para uma data válida

Isso mantém a API enxuta, mas ainda não equivale a um sistema completo de schema aninhado.

## Descrevendo um Schema

O método `describe()` retorna um resumo amigável para tooling e documentação.

```ts
console.log(memberSchema.describe());
```

Saída:

```ts
{
  fields: {
    _id: { ... },
    name: { ... },
    email: { ... },
    age: { ... },
    verified: { ... },
    __v: { ... }
  },
  timestamps: true,
  collection: undefined
}
```
