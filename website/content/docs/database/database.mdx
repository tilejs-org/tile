# Database

A classe `Database` é responsável por criar e gerenciar uma database do Tile.JS.

Ela define onde os arquivos serão armazenados, o nome da database, o comportamento do logger e outras configurações globais.

---

## Criando uma Database

A forma mais simples de criar uma database é utilizando todas as configurações padrão.

```ts title="src/index.ts"
import { Database } from "@tile.js/database";

const database = new Database();
```

Por padrão, será criada uma database chamada **test**, armazenada em:

```text
.tile/database/test
```

---

## Configuração completa

Todas as opções podem ser personalizadas.

```ts title="src/index.ts"
import { Database } from "@tile.js/database";

const database = new Database({
  dbName: "my-database",

  storage: "workspace",

  compression: false,

  logger: {
    enabled: true,
    colors: true,
  },
});
```

## Modos de armazenamento

### Workspace

Salva os dados dentro do projeto.

```text
.tile/
└── database/
    └── test/
```

Ideal para aplicações, APIs e projetos que mantêm seus dados junto ao código-fonte.

---

### Global

Salva os dados no diretório do usuário.

```text
~/.tile/
└── database/
    └── test/
```

Esse modo é recomendado para CLIs e ferramentas utilizadas em diversos projetos.

---

## Logger

O logger facilita o desenvolvimento exibindo informações importantes durante a execução.

```ts
import { Database } from "@tile.js/database";

const database = new Database({
  logger: {
    enabled: true,
    colors: true,
  },
});
```

Saída:

```bash
[DEBUG] Database initialized
[DEBUG] Collection "users" created
```

## Estrutura criada

A Database cria automaticamente toda a estrutura necessária.

<Files>
  <Folder name=".tile">
    <Folder name="database" defaultOpen>
      <Folder name="my-database" defaultOpen />
    </Folder>
  </Folder>
</Files>

Você não precisa criar nenhuma pasta manualmente.
