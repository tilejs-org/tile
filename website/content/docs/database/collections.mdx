# Collections

Uma **Collection** é a interface que você usa para trabalhar com documentos.

Toda collection pertence a uma `Database` e utiliza um `Schema` para validar e normalizar os dados.

<Callout type="info">
Uma collection é parecida com uma tabela em bancos relacionais, mas armazena documentos em vez de linhas fixas.
</Callout>

---

## Criando uma Collection

Primeiro crie um schema.

```ts title="schema.ts"
import { Schema } from "@tile.js/database";

interface Member {
  _id?: string;
  name: string;
  email: string;
}

export const memberSchema = new Schema<Member>({
  name: {
    type: String,
    required: true,
  },
  email: {
    type: String,
    unique: true,
  },
});
```

Depois registre a collection na database.

```ts title="database.ts"
import { Database } from "@tile.js/database";
import { memberSchema } from "./schema";

const connection = new Database();

export const database = Object.assign(connection, {
  members: connection.collection<Member>("members", memberSchema),
});
```

A classe `Database` também expõe `model()` como alias de `collection()`.

## Exemplo

```ts
const member = await database.members.create({
  name: "Israel",
  email: "israel@gmail.com",
});

console.log(member);
```

Buscar um documento:

```ts
const member = await database.members.findOne({
  email: "israel@gmail.com",
});
```

Atualizar:

```ts
const updated = await database.members.updateOne(
  {
    email: "israel@gmail.com",
  },
  {
    name: "Israel Rabbi",
  },
);
```

Remover:

```ts
await database.members.deleteOne({
  email: "israel@gmail.com",
});
```


## Organização recomendada

```text
src/
├── database.ts
├── schemas/
│   └── member.ts
└── index.ts
```

Separar `schemas` e `database.ts` já resolve bem a maior parte dos projetos pequenos e médios.
