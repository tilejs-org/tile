# Bem-vindo

O **@tile.js/database** é uma camada de persistência tipada para aplicações **Node.js** e **Bun**.

A lib continua com o adapter local padrão em BSON, mas também suporta adapters externos para **MongoDB** e **Mongoose**.

<Callout type="info">
Se esta é sua primeira vez utilizando o Tile.JS Database, siga esta ordem de leitura:

**Instalação → Database → Adapters → Schema → Collections → CRUD**
</Callout>

---

## Exemplo rápido

```ts title="src/index.ts"
import { Database, Schema } from "@tile.js/database";

interface Member {
  _id?: string;
  email: string;
  name: string;
}

const memberSchema = new Schema<Member>({
  email: { type: String, required: true, unique: true, lowercase: true },
  name: { type: String, required: true, trim: true },
});

const connection = new Database({
  dbName: "example-project",
});

export const database = Object.assign(connection, {
  members: connection.collection<Member>("members", memberSchema),
});

await database.connect();

const member = await database.members.create({
  email: "ISRAEL@GMAIL.COM",
  name: " Israel ",
});

console.log(member);

await database.disconnect();
```

<Callout type="tip">
No modo padrão, a estrutura de diretórios, os índices e o `_schema.json` são criados automaticamente na primeira utilização da collection.
</Callout>

  type={{

    },

      description: "Compacta os arquivos BSON com Gzip no adapter padrão.",
      type: "boolean",
      default: "false",
    },
  }}
/>

<Callout type="warning">
Quando `storage` recebe um adapter customizado, opções como `dbName` e `compression` deixam de ser válidas no `new Database(...)` e devem ser configuradas no próprio adapter.
</Callout>

## Estrutura criada pelo adapter padrão

Após criar a primeira collection local, a estrutura do projeto ficará semelhante a esta:

<Files>
  <Folder name=".tile">
    <Folder name="database" defaultOpen>
      <Folder name="test" defaultOpen>
        <Folder name="members" defaultOpen>
          <File name="_collection_index_id.json" />
          <File name="_collection_index_email.json" />
          <File name="_schema.json" />
          <File name="*.bson" />
        </Folder>
      </Folder>
    </Folder>
  </Folder>

  <Folder name="src" defaultOpen>
    <File name="index.ts" />
  </Folder>

  <File name="package.json" />
  <File name="tsconfig.json" />
</Files>

<Callout type="info">
A pasta <code>.tile</code> só existe para o adapter padrão local. Adapters externos como MongoDB e Mongoose usam a infraestrutura fornecida pelo consumidor.
</Callout>
