# Instalação

O **@tile.js/database** pode ser instalado utilizando qualquer gerenciador de pacotes compatível com o Node.js ou Bun.

<Callout type="info">
É recomendado utilizar a versão mais recente do pacote para aproveitar novos recursos e correções.
</Callout>

---

## Instale o pacote

<Tabs items={["bun", "npm", "pnpm", "yarn"]} >

<Tab value="bun">

```bash
bun add @tile.js/database
```

</Tab>

<Tab value="npm">

```bash
npm install @tile.js/database
```

</Tab>

<Tab value="pnpm">

```bash
pnpm add @tile.js/database
```

</Tab>

<Tab value="yarn">

```bash
yarn add @tile.js/database
```

</Tab>

</Tabs>

---

## Requisitos

| Requisito | Versão |
| ---------- | ------ |
| Node.js | >= 24.x |
| Bun | >= 1.x |
| TypeScript | >= 5.x |

---

## Verificando a instalação

Crie um arquivo chamado `index.ts` e importe o pacote:

```ts title="index.ts"
import { Database, Schema } from "@tile.js/database";

console.log("Tile.JS Database instalado com sucesso!");
```

Execute:

<Tabs items={["bun", "node"]}>

<Tab value="bun">

```bash
bun index.ts
```

</Tab>

<Tab value="node">

```bash
node index.ts
```

</Tab>

</Tabs>

Se nenhuma mensagem de erro aparecer, a instalação foi concluída corretamente.

## O que é criado?

Na primeira vez que uma coleção é criada, o Tile.JS gera automaticamente toda a estrutura necessária para armazenar os dados.

Isso inclui:

- A pasta `.tile`
- A database
- As collections
- Os arquivos de schema
- Os índices únicos (`unique`)

Você não precisa criar nenhum desses arquivos manualmente.

<Callout type="tip">
O Tile.JS utiliza arquivos **BSON** para armazenar documentos e arquivos **JSON** para metadados e índices, permitindo um banco de dados leve e portátil.
</Callout>
